# δme-13.exe - Scepter Bootstrap

## 項目描述

這是一個模擬 δme-13.exe（權杖 δme-13）的程序，實現了翁法羅斯模擬在權杖上以「週期」重複運行的機制。當遭「毀滅」窺見後會發生異常並回捲，形成輪迴效果。

## 功能特點

- **權杖啟動序列**：模擬 δme-13 權杖的初始化過程
- **進度顯示**：實時顯示各個階段的執行進度
- **故障檢測**：在「觀測同步」階段有 35% 機率觸發「毀滅觀測」造成的計算漂移
- **輪迴機制**：當發生 IRONTOMB_TRANSIENT_STATE 錯誤時，系統會回捲並重啟同一週期
- **日誌記錄**：所有操作都會記錄到 `As-Ive-Written.log` 文件中

## 運行方式

### 方法 1：使用批處理文件（推薦）

```bash
δme-13.bat
```

### 方法 2：直接運行 Python 腳本

```bash
python delta_me13_sim.py
```

## 系統要求

- Python 3.9 或更高版本
- Windows 系統（批處理文件）

## 程序流程

1. **啟動序列**：

   - 初始化權杖核心 (Celestial-Body Neuron)
   - 載入翁法羅斯基線資料 (Baseline)
   - 掛載記事介面 (Journal UI)
   - 連接命途模組

2. **週期執行**：

   - 拓撲重建（地貌/城邦/族譜）
   - 敘事索引綁定（'如我所書'）
   - 命途執行紀錄寫入（七命途）
   - 觀測同步（星神側—觀者效應）
   - 結果收斂與封存

3. **故障處理**：
   - 當檢測到 DESTRUCTION_GAZE 時觸發計算漂移
   - 漂移校準失敗時產生 IRONTOMB_TRANSIENT_STATE 錯誤
   - 系統回捲到檢查點並重啟同一週期

## 輸出文件

- `As-Ive-Written.log`：包含所有操作的詳細日誌記錄

## 自定義設置

可以在 `delta_me13_sim.py` 中調整以下參數：

- `max_loops`：最大輪迴次數（默認 8）
- 各階段的錯誤觸發機率
- 進度條的顯示速度和樣式

## 遊戲背景

此程序基於《崩壞：星穹鐵道》中的設定：

- **翁法羅斯**：被模擬的世界
- **δme-13**：權杖編號
- **As I've Written**：記事介面
- **Irontomb**：封印系統
- **毀滅**：星神之一，其觀測會造成系統不穩定
